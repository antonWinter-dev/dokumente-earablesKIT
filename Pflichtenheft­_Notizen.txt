Pflichtenheft:

Zielbestimmung:
Musskriterien
o	Bluetooth Verbindung
o	Bibliothek für Schritterkennung (Frequenz)
o	Gerätedaten auslesen (parsen, etc.)
o	Cross-Plattform GUI Anwendung
(o   Open-Source im Sinne der MIT Lizenz)

Wunschkriterien
o	Daten visualisieren
o	Situps/Liegestützen erkennen
o	Sprachanweisung in Bezug auf Trainingsprogramm

Abgrenzungskriterien
o	Musik nicht veränderbar
o	Datenspeicherung nur auf dem Device
o	Bewusstes Austricksen der Messungen werden nicht beachtet
o   Nur von Xamarin unterstütze Geräte werden unterstützt

Funktionale Anordnungen
-	Der Benutzer muss die Parameter der Messung konfigurieren können
-	Der Benutzer kann seine Einstellungen als Datei exportieren/importieren
-	Der Benutzer kann Liegestützen/Situps für einen zählen lassen

Nichtfunktionale Anforderungen
-	Die GUI muss ansprechend wirken
-	Package lost behandeln?


Produktdaten:
-	Die Trainingsdaten werden von der Anwendung auf dem Gerät gespeichert  

GUI:
1.	Messdaten anzeigen
2.	Setting Website
3.	Lauf tracken
4.	Situps/Liegestützen zählen
5.	Trainingsprogramm per Text to Speech

Definitionen von Begriffen 
-   statt Echtzeit lieber "live" verwenden, live := mit einer Verzögerung von maximal 3 Sekunden


Fragen an Kunden:
-	Wie genau Schritterkennung? Echtzeit?
-	Können wir die Funktionalität flexibler gestalten?
-	Müssen wir die Musik anpassen können? Essenziell?
-	Nichtfunktionale Anforderungen? Welche genau? Woher kann man die Realisierbarkeit abschätzen? Wie lange muss es dauern?
-	Was genau ist das Architekturdiagramm
-	Kann man die Struktur des Lastenheftes verändern?
